# ようこそ — このコースの歩き方

データ基盤づくりは、ダムの建設に似ています。水（データ）をただ貯めるだけなら誰でもできます。難しいのは、必要なときに必要な人へ、きれいな水を確実に届け続けること。そして年月が経っても壊れずに使われ続けることです。このコースは、その「壊れない」を実現する考え方と技術を、最初の一歩から渡します。

## このコースは誰のためか

特別な前提知識はいりません。SQLをなんとなく読めれば十分です。これからデータ基盤に関わるエンジニア、データを扱うアナリスト、あるいは「データ基盤って結局なに？」という人。誰でも入口に立てるよう、専門用語は初めて出てきたところで必ず噛み砕いて説明します。

## 何を学べるのか

このコースの主役は「腐らないデータ基盤」です。データ基盤は作って終わりではありません。むしろ作った後にこそ本当の戦いが始まります。テーブルを設計し、データを流し込み、分析できる形に整える——その一連の流れを、共通のサンプルドメイン（顧客・商品・注文・イベント）を題材に、手を動かしながら学びます。

:::insight 腐らないデータ基盤とは
「使い続けられる」基盤のこと。技術的に正しいだけでは足りません。人に発見され、信頼され、安全に使われ、変化に耐える——この4つがそろって初めて基盤は生き続けます。
:::

## "腐る" 4つの失敗モード

データ基盤が腐るとき、その失敗はだいたい次の4パターンに収まります。各レッスンの終盤では、その内容がどの失敗モードを防ぐのかを必ず明示します。

1. 作ったけど使われない（unused）——発見されず、信頼されず、誰も触らない。
2. 価値がチームに閉じる（siloed）——作った人しか使えず、知識が属人化する。
3. 想定外の使い方をされる（misused）——定義や粒度が曖昧で、誤った数字が独り歩きする。
4. 使われすぎて変更できない（ossified）——みんなが依存しすぎて、もう直せない。

この4つを頭の片隅に置くだけで、設計の質は大きく変わります。

## 進め方とPlayground

各レッスンは「比喩で直感をつかむ → 正確な定義 → SQLや図で具体化 → よくある失敗 → 腐らせないポイント → 演習」という流れで進みます。途中に出てくるSQLは、付属のPlaygroundでそのまま実行できます。Playgroundには上記サンプルドメインのテーブルが用意してあるので、読むだけでなく、自分の手で叩いて確かめてください。理解の定着がまるで違います。


## 修了後にできること

このコースを終えると、あなたは生データを分析しやすいスター・スキーマへ整える流れを理解し、なぜその設計が「腐りにくい」のかを言葉で説明できるようになります。単に動くものではなく、半年後・一年後の自分やチームに感謝される基盤を、自分の手で設計・実装できる——それがゴールです。

それでは、最初の一歩を踏み出しましょう。

## まとめ

- このコースの目的は「腐らないデータ基盤」を自分で設計・実装できるようになること。
- 前提知識は最小限。SQLが少し読めれば誰でも始められる。
- 基盤が腐る失敗モードは4つ（unused / siloed / misused / ossified）。各レッスンで明示的に対策を学ぶ。
- 学びの流れは「直感 → 定義 → 具体 → 失敗例 → 対策 → 演習」。Playgroundで必ず手を動かす。
